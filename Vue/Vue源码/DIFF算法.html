DIFF算法

虚拟DOM的概念：使用js对象去表示DOM节点，DOM中的所有属性都可以对应相应的虚拟DOM属性。

真实DOM

 <div class="box">
    <h3>我是一个标题</h3>
    <ul>
      <li>111</li>
      <li>222</li>
      <li>333</li>
    </ul>
  </div>

虚拟DOM

 {
    sel: 'div',
    data: {
        class: { box: true },
    },
    children: [
        {
            sel: 'h3',
            data: {},
            text: '我是一个标题',
        },
        {
            sel: 'ul',
            data: {},
            children: [
                { sel: 'li', data: {}, text: '111'},
                { sel: 'li', data: {}, text: '222'},
                { sel: 'li', data: {}, text: '333'},
            ],
        },
    ],
}

Snabbdom环境搭建

snabbdom是著名的虚拟DOM库，是diff算法的鼻祖，Vue源码借鉴了snabbdom；

官方git：https://github.com/snabbdom/snabbdom

在git上的snabbdom源码是用TypeScript写的，git上并不提供编译好的JavaScript版本；如果要直接使用build出来的JavaScript版的snabbdom库，可以从npm上下载：

npm i -D snabbdom

使用h函数 和 patch 函数

H函数成员

h函数的使用方式

没有子节点的

const myVnode = h('a', { props: { href: 'www.baidu.com' } }, '百度')

包含一个子节点的

h('sel',{},h())

包含多个子节点的

h('sel',{},[])

h函数的作用

h函数是将编译好的模板生成相应的虚拟DOM。

h函数对传入的参数由三个部分组成，sel：标签 数据、属性：data 子级或文本内容：text

h函数实际是对参数的不同类型进行判断，分别调用vnode函数（函数的重载）产生不同的虚拟DOM

vnode函数将传入的参数以对象的形式返回，如果data属性里有key，那么将key也添加到返回的对象中

patch函数

Diff算法主要在patch函数内实现，作用是判断旧的虚拟DOM 和 新的虚拟DOM

patch函数由init函数返回，patch函数在源码中返回一个新的虚拟DOM

先判断旧的节点是不是一个虚拟DOM（如果传入container容器的节点就是真实的DOM），如果不是虚拟DOM就调用一个函数将真实的DOM包装成虚拟DOM

如何判断两个节点是不是同一个节点呢？

是否是同一个key值

sel属性是否相同

如果是相同的节点

是进行一个精细化的过程，在patchVnode函数中进行，这是DIFF算法的核心

不是相同的节点

直接创建新的节点，删除旧的节点
